# Glossary

## Purpose
Define common terms used across product, engineering, governance, and operations documentation.

## Terms
- **Adapter Layer**: Provider-specific integration boundary that maps external source data into internal normalized format.
- **Approved Item**: Content item reviewed by admin and allowed for public display.
- **Link-First Strategy**: Data policy storing only metadata references and source URLs, not media binaries.
- **Manual Refresh**: Member-triggered content sync for an approved linked account.
- **Moderation Queue**: Admin workflow where pending links and pending items are approved or rejected.
- **Normalized Content Item**: Unified internal representation across providers (e.g., title, canonical URL, provider, tags).
- **Provider Capability Matrix**: Documentation of source-specific auth, feed access, rate limits, and failure modes.
- **RLS (Row-Level Security)**: Database policy restricting data access based on user role and row ownership.
- **Wave Authoring**: Priority-based staged documentation rollout (Wave 1, 2, 3).
