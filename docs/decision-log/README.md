# Decision Log

## Purpose
Record architecture, policy, and workflow decisions with rationale and consequences.

## Usage
- Add one ADR file per material decision.
- File naming: `000X-short-title-adr.md`.
- Keep entries immutable; supersede with a new ADR instead of rewriting history.

## Required ADR Sections
- Context
- Decision
- Consequences
- Alternatives Considered
- Date and Owner
